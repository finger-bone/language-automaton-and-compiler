Non-Deterministic Finite Automaton With Empty Transitions

We have nullable regular languages, and so we have nullable NFA, that is, NFA with empty transitions, also called e-NFA (ϵ\epsilonϵ-NFA).

Components of e-NFA​

Components​

e-NFA has all its parts the same as NFA. But we allow instead of making,

T^({qi},λ as empty sentence)={qi}\hat{\mathcal{T}}(\{q_i\}, \lambda \text{ as empty sentence}) = \{q_i\}T^({qi​},λ as empty sentence)={qi​}

We would allow to accept λ\lambdaλ as a symbol,

T^({qi},λ as a sentence with one symbol)={qj}\hat{\mathcal{T}}(\{q_i\}, \lambda \text{ as a sentence with one symbol}) = \{q_j\}T^({qi​},λ as a sentence with one symbol)={qj​}

Please note that, when it comes to T^\hat{\mathcal{T}}T^, it is the NFA that accepts λ\lambdaλ as a normal symbol instead of the empty sentence. But for e-NFA, we use T\mathcal{T}T and it treats λ\lambdaλ an empty sentence.

Because (λ as empty sentence)n=λ as empty sentence  (n≥0)(\lambda \text{ as empty sentence})^n = \lambda \text{ as empty sentence} \; (n \geq 0)(λ as empty sentence)n=λ as empty sentence(n≥0), thus,

T({qi},v)=⋃n≥0,m≥0T({qi},(λ as empty sentence)nv(λ as empty sentence)m)=⋃n≥0,m≥0T^(T^(T^({qi},(λ as a symbol)n),v),(λ as a symbol)m)\mathcal{T}(\{q_i\}, v) = \bigcup_{n \geq 0, m \geq 0} \mathcal{T}(\{q_i\}, (\lambda \text{ as empty sentence})^nv(\lambda \text{ as empty sentence})^m) \\ = \bigcup_{n \geq 0, m \geq 0} \hat{\mathcal{T}}(\hat{\mathcal{T}}(\hat{\mathcal{T}}(\{q_i\}, (\lambda \text{ as a symbol})^n),v), (\lambda \text{ as a symbol})^m)T({qi​},v)=n≥0,m≥0⋃​T({qi​},(λ as empty sentence)nv(λ as empty sentence)m)=n≥0,m≥0⋃​T^(T^(T^({qi​},(λ as a symbol)n),v),(λ as a symbol)m)

This seems kind of complex. We can define a concept called the lambdalambdalambda closure of a set of states, the Qiλ\mathcal{Q_i}^\lambdaQi​λ,

Qiλ=⋃n≥0T^(Qi,λn)\mathcal{Q_i}^\lambda = \bigcup_{n \geq 0} \hat{\mathcal{T}}(\mathcal{Q_i}, \lambda^n)Qi​λ=n≥0⋃​T^(Qi​,λn)

That is to say, the lambdalambdalambda closure of a set of states is the set of states that can be reached by a sequence of empty transitions.

So the e-NFA transition function is,

T(Qi,v)=T^(Qiλ,v)λ\mathcal{T}(\mathcal{Q_i}, v) = \hat{\mathcal{T}}(\mathcal{Q_i}^\lambda, v)^\lambdaT(Qi​,v)=T^(Qi​λ,v)λ

Much easier to understand.

Removing the Empty Transitions​

e-NFA is more easier to design compared to NFA. And we need a way to convert it to NFA, so that it can be made into a minimized DFA. For the minimized DFA, we can write the most efficient program to recognize the language.

How to remove the empty transitions? Let's consider the transition table of the e-NFA,

Grammar of e-NFA​

We can take a similar rule to convert from e-NFA to its grammar. So in the end, we will have,

A→aAA→aA→BA \rightarrow aA \\ A \rightarrow a \\ A \rightarrow BA→aAA→aA→B

Where A→BA \rightarrow BA→B is corresponding to the empty transition.

Because all A→BA \rightarrow BA→B can be reduced into,

A→aAA→aA \rightarrow aA \\ A \rightarrow a \\A→aAA→a

We simply find the generated result of BBB that is in form of bBbBbB or bbb. Then add the new grammar to replace the old one,

A→aBA→bA \rightarrow aB \\ A \rightarrow b \\A→aBA→b

So that we can get a regular grammar.

Please note that we do not need to consider the case of,

A→bB∣cCA \rightarrow bB | cCA→bB∣cC

Because e-NFA allows non-deterministic transitions.

Thus, e-NFA also accepts regular language.
