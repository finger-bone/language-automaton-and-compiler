Non-Deterministic Finite Automaton

Non-Deterministic Finite automaton (NFA) is an extension to the DFA that is simpler to design. It can be converted into a DFA and thus, it also accepts a regular language.

Components of NFA​

Components​

A NFA has the following components,

  • State set Q={qs,q1,q2,⋯ ,qn}\mathcal{Q} = \{qs, q_1, q_2, \cdots, q_n\}Q={qs,q1​,q2​,⋯,qn​}
  • A vocabulary set V\mathcal{V}V
  • A transition table T(q,v)\mathcal{T}(q, v)T(q,v)
  • A start state {qs}\{ qs \}{qs}
  • An accept state set F={qf,qf1,qf2,⋯ ,qfn}\mathcal{F} = \{q_f, q_{f1}, q_{f2}, \cdots, q_{fn}\}F={qf​,qf1​,qf2​,⋯,qfn​}

Unlike DFA, the transition table for NFA yields a set instead of a single state. In addition, the current state of NFA is also a set instead of a single state.

For the transition function, we extend it to take a set of states, that is,

T(⋃i∈Iqi,v)=⋃i=IT(qi,v)\mathcal{T}(\bigcup_{i\in \mathbb{I}} q_i, v) = \bigcup_{i=\mathbb{I}} \mathcal{T}(q_i, v)T(i∈I⋃​qi​,v)=i=I⋃​T(qi​,v)

Where I\mathbf{I}I is any set of indices.

This means that, unlike DFA, NFA has the capacity at staying at multiple states at the same time.

In the end, the accept condition is,

T({qs},v)∩F≠∅\mathcal{T}(\{q_s\}, v) \cap \mathcal{F} \neq \emptysetT({qs​},v)∩F=∅

Example​

We consider the following example,

Let's test it on 001001001, first do it graphically,

After taking in 000, both q0q0q0 and q1q1q1 are in the current state set,

After taking another 000, the current state set remains unchanged,

After taking 111, only q1q1q1 is in the current state set,

Because there exists a accept state in the final state set, we accept the string.

Let's consider what would happen with 001000100010. Obviously, we can continue from before.

Because q1q1q1 transits to an empty set after taking in 000, so, now the current state set is ∅\emptyset∅.

As we reached the end of the string, there exists no accept state in the current state set, so we reject the string.

Please note that, when the current state set is empty, no matter how many more words are going in, it will remain empty. This is what we call the dead state. If the prefix of a test string will lead the automaton into the dead state, we can early reject it.

Converting NFA to DFA​

Based on the previous example, we can tell that, if we use the subset of Q\mathcal{Q}Q as the state of a new automaton, we will have the corresponding DFA automatontically. However, if we just list all subset one by one, there will be too many states. Although you can minimize that DFA, it's still hard to convert considering the exponential complexity.

So we have a way to iterate through all possible subset of Q\mathcal{Q}Q, and we call it the Kleene's algorithm.

The Grammar of NFA​

NFA Grammar is Regular Grammar​

With the same method as the DFA, we can have the following rules.

A→aBA→aA \rightarrow aB \\ A \rightarrow aA→aBA→a

This is exactly the regular grammar. The grammar of NFA is the regular grammar, the regular grammar is the grammar of NFA. And unlike DFA that poses extra constraints, NFA does not pose any constraints. The grammar of NFA is the regular grammar.

From Regular Grammar to NFA​

For a regular grammer, it takes either of the forms,

A→aBA \rightarrow aBA→aB

Or,

A→aA \rightarrow aA→a

If it is the first case, then we create two states AAA and BBB if there doesn't exist one. Then adding an edge with label aaa from AAA to BBB.

If it's the second case, we create a state AAA and an accept state qaqaqa if there doesn't exist one. Then we add an edge from AAA to qaqaqa with label aaa.

By constructing the automaton, we can test all strings in the given regular language.

This is because, for any sentence vvv generated by the regular grammar.

S⇒∗vS \Rightarrow^* vS⇒∗v

Let's break it into,

S⇒v1⇒v2…⇒vS \Rightarrow v_1 \Rightarrow v_2 \ldots \Rightarrow vS⇒v1​⇒v2​…⇒v

Then for each intermediate sentence viv_ivi​,

T(qs,0@v)=q1\mathcal{T}(qs, 0@v) = q_1T(qs,0@v)=q1​

Then,

T(qi,i@v)=qi+1\mathcal{T}(q_i, i@v) = q_{i + 1}T(qi​,i@v)=qi+1​

So there exists a one to one map from,

T(qi,i@v)=qi+1\mathcal{T}(q_i, i@v) = q_{i + 1}T(qi​,i@v)=qi+1​

To,

vi⇒vi+1v_i \Rightarrow v_{i + 1}vi​⇒vi+1​

Thus, derivation is state transition. And thus, we can prove that all regular languages can be represented by NFA.

And because NFA cna be converted to a DFA, we can prove that all regular languages can be represented by DFA.
