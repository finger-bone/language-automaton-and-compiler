LL(1) Grammar

For some unambiguous language, there is a way that we can scan the sentence from left to right one by one, and find the only possible derivation based on the next one character (if there exists two, it's ambiguous). This is called an LL(1)LL(1)LL(1) grammar. The first LLL stands for left starting. The second LLL stands for leftmost derivation. 111 means that the grammar only need to look ahead of one symbol to know which rule to apply.

This part, we will focus on how we can test wether a CFG is LL(1)LL(1)LL(1), and how we can build a parse tree for an LL(1)LL(1)LL(1) language.

Testing a CFG for LL(1)LL(1)LL(1)​

Eliminate Left Recursion​

Left recursion refers to the rules in form of,

A→AaA \rightarrow AaA→Aa

Or there may exists indirect left recursion,

A→BbB→AaA \rightarrow Bb \\ B \rightarrow AaA→BbB→Aa

If A⇒AxA \Rightarrow AxA⇒Ax can be inferred from the given rule set, the given rule set exists left recursion.

All CFG that has left recursion can't be LL(1)LL(1)LL(1). However, we can eliminate left recursion by rewriting. That is,

Rewrite Left Recursion​

A→Aα∣βA \rightarrow A \alpha | \betaA→Aα∣β

To eliminate the left recursion,

  1. Create a new non-terminal (let’s call it A′A'A′) to handle the recursive part.
  2. Reformulate the rules so that the recursive calls happen in the right-hand side and are postponed.

That is, making it, into:

A→βA’A \rightarrow \beta A’A→βA’ A’→αA’∣λA’ \rightarrow \alpha A’ | \lambdaA’→αA’∣λ

So that it is right recursion instead of left recursion.

Flatten Indirect Left Recursion​

For indirect left recursion, we can flat them into direct left recursion. That is, for,

Ei→Aixi′∣ziA0→A1x1∣y1A1→A2x2∣y2…An→A0x0∣y0E_i \rightarrow A_ix'_i | z_i \\ A_0 \rightarrow A_1x_1 | y_1 \\ A_1 \rightarrow A_2x_2 | y_2 \\ \ldots \\ A_{n} \rightarrow A_0x_0 | y_0Ei​→Ai​xi′​∣zi​A0​→A1​x1​∣y1​A1​→A2​x2​∣y2​…An​→A0​x0​∣y0​

We should flatten this by iteratively substitution. That is, we can replace,

E1→A1x1′∣z1A0→A1x1∣y1A1→A2x2∣y2E_1 \rightarrow A_1x'_1 | z_1 \\ A_0 \rightarrow A_1x_1 | y_1 \\ A_1 \rightarrow A_2x_2 | y_2 \\E1​→A1​x1′​∣z1​A0​→A1​x1​∣y1​A1​→A2​x2​∣y2​

With,

A0→A2x2x1∣y2y1∣y1E1→A2x2x1′∣y2x1′∣z1A_0 \rightarrow A_2x_2x_1 | y_2y_1 | y_1 \\ E_1 \rightarrow A_2x_2x'_1 | y_2x'_1 | z_1 \\A0​→A2​x2​x1​∣y2​y1​∣y1​E1​→A2​x2​x1′​∣y2​x1′​∣z1​

This can be done iteratively until,

A0→A0x0xn…x1∣…A_0 \rightarrow A_0 x_0 x_n \ldots x_1 | \ldotsA0​→A0​x0​xn​…x1​∣…

So that there is no indirect left recursion.

Example​

S→A∣CA→Ab∣Ca∣aC→Ae∣fS \rightarrow A | C \\ A \rightarrow A b | C a | a \\ C \rightarrow A e | fS→A∣CA→Ab∣Ca∣aC→Ae∣f

We first flatten the indirect left recursion caused by,

A→CaC→AeA \rightarrow Ca \\ C \rightarrow A eA→CaC→Ae

So, we can eliminate CCC,

S→A∣Ae∣fA→Ab∣Aea∣f∣aS \rightarrow A | Ae | f \\ A \rightarrow A b | A e a | f | a \\S→A∣Ae∣fA→Ab∣Aea∣f∣a

Now there is no indirect left recursion.

Then we eliminate left recursion,

S→A∣Ae∣fA→fA′∣aA′∣fA′′∣aA′′A′→bA′∣λA′′→eA′′∣λS \rightarrow A | Ae | f \\ A \rightarrow fA' | aA' | fA'' | aA'' \\ A' \rightarrow bA' | \lambda \\ A'' \rightarrow eA'' | \lambda \\S→A∣Ae∣fA→fA′∣aA′∣fA′′∣aA′′A′→bA′∣λA′′→eA′′∣λ

This is the final grammar without left recursion.

Factor Common Prefixes​

Sometimes we may have,

A→xB∣xCA \rightarrow xB | xCA→xB∣xC

There are two targets with same prefix. We should factor them out. That is,

A→xA′A′→B∣CA \rightarrow xA' \\ A' \rightarrow B | CA→xA′A′→B∣C

This isn't present in our example.

Calculate the FIRST Set of Every Non-Terminal​

The FIRST set of a non-terminal is defined as,

FIRST(A)={0@v∣A⇒∗v}\text{FIRST}(A) = \{0@v | A \Rightarrow^* v \}FIRST(A)={0@v∣A⇒∗v}

If A⇒∗λA \Rightarrow^* \lambdaA⇒∗λ, then,

FIRST(A)={0@v∣A⇒∗v}+{λ}\text{FIRST}(A) = \{0@v | A \Rightarrow^* v \} + \{\lambda\}FIRST(A)={0@v∣A⇒∗v}+{λ}

We also define the FIRST set of a terminal as,

FIRST(a)={a}\text{FIRST}(a) = \{a\}FIRST(a)={a}

For a sentence xxx, the FIRST set is defined as,

FIRST(x)=FIRST(0@x)\text{FIRST}(x) = \text{FIRST}(0@x)FIRST(x)=FIRST(0@x)

That is, for xxx, FIRST(x)\text{FIRST}(x)FIRST(x) contains the first character of every possible derivation of xxx. For null sentence, we treat the first character as λ\lambdaλ.

Because we have eliminated left recursion, getting FIRST set is simple- we try every path of the derivation and, once we have a terminal symbol at the beginning, we add it to the set and end this path.

To be more specific, for each rule, if it's in form of,

A→awA \rightarrow awA→aw

So that aaa should be added to the FIRST set of AAA.

Or,

A→BwA \rightarrow BwA→Bw

So then every symbol in \text{FIRST}(B) - \{\lambda\}$$ should be added to the FIRST set of A$.

If B⇒∗λB \Rightarrow^* \lambdaB⇒∗λ and w→∗λw \rightarrow^* \lambdaw→∗λ, then we also add λ\lambdaλ to the FIRST set of AAA.

Sometimes, we also have,

A→λA \rightarrow \lambdaA→λ

We will add λ\lambdaλ to the FIRST set of AAA.

Because there is no left recursion, the second case won't case any problem.

For example,

S→A∣Ae∣fA→fA′∣aA′A′→bA′∣λS \rightarrow A | Ae | f \\ A \rightarrow fA' | aA' \\ A' \rightarrow bA' | \lambdaS→A∣Ae∣fA→fA′∣aA′A′→bA′∣λ

This is a subset of our previous left recursion elimination example. Thus there is also no left recursion.

We usually draw a graph that denotes the generated-by relationship between non-terminals. If a non-terminal AAA can be generated by BBB, we draw an edge from BBB to AAA.

We ignore self-loops.

So,

Then we usually do this in the order of topological sort. So that if we need to use the FIRST set of a non-terminal to calculate the FIRST set of another, they must have already been calculated.

tip

The topological order of a set of nodes in a graph is a linear ordering of the nodes such that for every edge u→vu\rightarrow vu→v in the graph, uuu comes before vvv in the ordering.

In practice, we take the first node that doesn't have any incoming edge as the start of the ordering. Then we remove this node and repeat the process until all nodes have been visited.

For acyclic graphs, we can always do thi.

So we start with A′A'A′, because the only rules for A′A'A′ are,

A′→bA′∣λA' \rightarrow bA' | \lambdaA′→bA′∣λ

So FIRST(A′)={b,λ}\text{FIRST}(A') = \{b, \lambda\}FIRST(A′)={b,λ}.

Then the AAA. Obviously, we can have,

FIRST(A)={f,a}\text{FIRST}(A) = \{f, a\}FIRST(A)={f,a}

Lastly for SSS, because SSS yields non-terminals AAA at the left, or terminal fff, so,

FIRST(S)={f}+FIRST(A)={f,a}\text{FIRST}(S) = \{f\} + \text{FIRST}(A) = \{f, a\}FIRST(S)={f}+FIRST(A)={f,a}

Calculate the FOLLOW Set of Every Non-Terminal​

The FOLLOW set of a non-terminal is defined as,

FOLLOW(A)={a∣v∈I,v=xAay}\text{FOLLOW}(A) = \{ a | v \in \mathcal{I}, v = xAay\}FOLLOW(A)={a∣v∈I,v=xAay}

Where aaa is the terminal symbol that is after AAA. xxx and yyy are arbitrary symbols.

That is to say, the FOLLOW set of a non-terminal is all the possible terminal symbols that are after the non-terminal during the derivation.

Specially, we will always add S\sout{S}S to the FOLLOW set of SSS. This is a special character denoting the end of the sentence. Because empty input is accepted, SSS can always have S\sout{S}S in the FOLLOW set.

Or alternatively, if you don't what to remember such an abrupt rule, you can add the following grammar,

S′→SSSS' \rightarrow \sout{S}S\sout{S}S′→SSS

Use S′S'S′ as the new starting symbol, then everything will play out naturally.

To get the FOLLOW set, more practically, we check every rule.

If it's in form of,

A→zBaxA \rightarrow zBaxA→zBax

Then we add aaa to the FOLLOW set of BBB (x,zx, zx,z is any intermediate sentence and aaa is a terminal symbol).

If it's in form of,

A→zBCxA \rightarrow zBCxA→zBCx

Then we add all of FIRST(C)\text{FIRST}(C)FIRST(C) to the FOLLOW set of BBB.

If it's in form of,

A→zBββ⇒∗λA \rightarrow zB\beta \quad \beta \Rightarrow^* \lambdaA→zBββ⇒∗λ

Then we add all of FOLLOW(A)\text{FOLLOW}(A)FOLLOW(A) to the FOLLOW set of BBB.

If,

A→aAsββ⇒∗λA \rightarrow aA s\beta \quad \beta \Rightarrow^* \lambdaA→aAsββ⇒∗λ

We ignore it because it doesn't contribute to the FOLLOW set of AAA.

Specially, we will always add S\sout{S}S to the FOLLOW set of SSS.

We still use,

S→A∣Ae∣fA→fA′∣aA′A′→bA′∣λS \rightarrow A | Ae | f \\ A \rightarrow fA' | aA' \\ A' \rightarrow bA' | \lambdaS→A∣Ae∣fA→fA′∣aA′A′→bA′∣λ

And do it in reversed topological order.

Previously, we have FIRST sets,

FIRST(S)={f,a}FIRST(A)={f,a}FIRST(A′)={b,λ}\text{FIRST}(S) = \{f, a\} \\ \text{FIRST}(A) = \{f, a\} \\ \text{FIRST}(A') = \{b, \lambda\}FIRST(S)={f,a}FIRST(A)={f,a}FIRST(A′)={b,λ}

There are no rules generating SSS, and because SSS is the start symbol, so, FOLLOW(S)={S}\text{FOLLOW}(S)=\{\sout{S}\}FOLLOW(S)={S}.

For AAA,

S→AS→AeS \rightarrow A \\ S \rightarrow AeS→AS→Ae

Thus, FOLLOW(A)=FOLLOW(S)+{e}={e,S}\text{FOLLOW}(A) = \text{FOLLOW}(S) + \{e\} = \{e, \sout{S} \}FOLLOW(A)=FOLLOW(S)+{e}={e,S}

For A′A'A′,

A→fA′∣aA′A′→bA′A \rightarrow fA' | aA' \\ A' \rightarrow bA'A→fA′∣aA′A′→bA′

Thus, FOLLOW(A′)=FOLLOW(A)={e,S}\text{FOLLOW}(A') = \text{FOLLOW}(A) = \{e, \sout{S} \}FOLLOW(A′)=FOLLOW(A)={e,S}.

Calculate the SELECT Set of Every Rule​

The SELECT set of a rule A→xA \rightarrow xA→x is defined as,

SELECT(A→x)={FIRST(x)∪FOLLOW(A)if λ∈FIRST(x)FIRST(x)if λ∉FIRST(x)\text{SELECT}(A \rightarrow x) = \begin{cases} \text{FIRST}(x) \cup \text{FOLLOW}(A) & \text{if } \lambda \in \text{FIRST}(x) \\ \text{FIRST}(x) & \text{if } \lambda \not\in \text{FIRST}(x) \end{cases}SELECT(A→x)={FIRST(x)∪FOLLOW(A)FIRST(x)​if λ∈FIRST(x)if λ∈FIRST(x)​

SELECT(A→x)\text{SELECT}(A \rightarrow x)SELECT(A→x) is all the possible terminal symbols that can exist in a final sentence that can be generated from the rule.

Previously, we had,

FIRST(S)={f,a}FOLLOW(S)={S}FIRST(A)={f,a}FOLLOW(A)={e,S}FIRST(A′)={b,λ}FOLLOW(A′)={e,S}\text{FIRST}(S) = \{f, a\} \\ \text{FOLLOW}(S) = \{\sout{S}\} \\ \text{FIRST}(A) = \{f, a\} \\ \text{FOLLOW}(A) = \{e, \sout{S}\} \\ \text{FIRST}(A') = \{b, \lambda\} \\ \text{FOLLOW}(A') = \{e, \sout{S}\}FIRST(S)={f,a}FOLLOW(S)={S}FIRST(A)={f,a}FOLLOW(A)={e,S}FIRST(A′)={b,λ}FOLLOW(A′)={e,S}

And rules,

S→A∣Ae∣fA→fA′∣aA′A′→bA′∣λS \rightarrow A | Ae | f \\ A \rightarrow fA' | aA' \\ A' \rightarrow bA' | \lambdaS→A∣Ae∣fA→fA′∣aA′A′→bA′∣λ

So,

SELECT(S→A)=FIRST(A)={f,a}SELECT(S→Ae)=FIRST(A)={f,a}SELECT(A→fA′)=FIRST(fA′)={f}SELECT(A→aA′)=FIRST(aA′)={a}SELECT(A′→bA′)=FIRST(bA′)+FOLLOW(A′)={b,e,S}SELECT(A′→λ)=FIRST(λ)={λ}\text{SELECT}(S \rightarrow A) = \text{FIRST}(A) = \{f, a\} \\ \text{SELECT}(S \rightarrow Ae) = \text{FIRST}(A) = \{f, a\} \\ \text{SELECT}(A \rightarrow fA') = \text{FIRST}(fA') = \{f\} \\ \text{SELECT}(A \rightarrow aA') = \text{FIRST}(aA') = \{a\} \\ \text{SELECT}(A' \rightarrow bA') = \text{FIRST}(bA') + \text{FOLLOW}(A') = \{b, e, \sout{S}\} \\ \text{SELECT}(A' \rightarrow \lambda) = \text{FIRST}(\lambda) = \{\lambda\}SELECT(S→A)=FIRST(A)={f,a}SELECT(S→Ae)=FIRST(A)={f,a}SELECT(A→fA′)=FIRST(fA′)={f}SELECT(A→aA′)=FIRST(aA′)={a}SELECT(A′→bA′)=FIRST(bA′)+FOLLOW(A′)={b,e,S}SELECT(A′→λ)=FIRST(λ)={λ}

Test for LL(1)LL(1)LL(1)​

For every rule in form of,

A→x0∣x1∣…∣xnA \rightarrow x_0 | x_1 | \ldots | x_nA→x0​∣x1​∣…∣xn​

If

SELECT(A→x0)∩SELECT(A→x1)∩…∩SELECT(A→xn)≠∅\text{SELECT}(A\rightarrow x_0) \cap \text{SELECT}(A\rightarrow x_1) \cap \ldots \cap \text{SELECT}(A\rightarrow x_n) \neq \emptysetSELECT(A→x0​)∩SELECT(A→x1​)∩…∩SELECT(A→xn​)=∅

Then the grammar is not LL(1)LL(1)LL(1).

In this example,

SELECT(S→A)=FIRST(A)={f,a}SELECT(S→Ae)=FIRST(A)={f,a}SELECT(A→fA′)=FIRST(fA′)={f}SELECT(A→aA′)=FIRST(aA′)={a}SELECT(A′→bA′)=FIRST(bA′)+FOLLOW(A′)={b,e,S}\text{SELECT}(S \rightarrow A) = \text{FIRST}(A) = \{f, a\} \\ \text{SELECT}(S \rightarrow Ae) = \text{FIRST}(A) = \{f, a \} \\ \text{SELECT}(A \rightarrow fA') = \text{FIRST}(fA') = \{f\} \\ \text{SELECT}(A \rightarrow aA') = \text{FIRST}(aA') = \{a\} \\ \text{SELECT}(A' \rightarrow bA') = \text{FIRST}(bA') + \text{FOLLOW}(A') = \{b, e, \sout{S}\}SELECT(S→A)=FIRST(A)={f,a}SELECT(S→Ae)=FIRST(A)={f,a}SELECT(A→fA′)=FIRST(fA′)={f}SELECT(A→aA′)=FIRST(aA′)={a}SELECT(A′→bA′)=FIRST(bA′)+FOLLOW(A′)={b,e,S}

S→AS \rightarrow AS→A and S→AeS \rightarrow AeS→Ae generates overlapping SELECT sets, so the grammar is not LL(1)LL(1)LL(1).

Constructing LL Table​

Now we need to create a table based on the FIRST and FOLLOW sets. This table can tell us what rule we should use to derive a sequence.

The table has row indices of every non-terminal, and column indices of every terminals. We note this table as, LLTb(A,a)\mathrm{LLTb}(A, a)LLTb(A,a).

  • For every A→xA \rightarrow xA→x, if ∃a∈FIRST(x)\exist a \in FIRST(x)∃a∈FIRST(x), such that a=0@xa = 0 @ xa=0@x or a∈FIRST(0@x)a \in \text{FIRST}(0 @ x)a∈FIRST(0@x), then we add A→xA \rightarrow xA→x to the table cell LLTb(A,a)\mathrm{LLTb}(A, a)LLTb(A,a).
  • For every A→xA \rightarrow xA→x, if x=λx = \lambdax=λ or λ∈FIRST(x)\lambda \in \text{FIRST}(x)λ∈FIRST(x), for each b∈FOLLOW(A)b \in \text{FOLLOW}(A)b∈FOLLOW(A), we add A→xA \rightarrow xA→x to the table cell LLTb(b,a)\mathrm{LLTb}(b, a)LLTb(b,a).

Do both checks for every rule.

Based on the given rules, we create the LL table,

For S→dS \rightarrow dS→d, d∈FIRST(d)d \in \text{FIRST}(d)d∈FIRST(d), so we add S→dS \rightarrow dS→d to the table cell LLTb(S,d)\mathrm{LLTb}(S, d)LLTb(S,d).

For S→AS \rightarrow AS→A, f∈FIRST(A)f \in \text{FIRST}(A)f∈FIRST(A), so we add S→AS \rightarrow AS→A to the table cell LLTb(S,f)\mathrm{LLTb}(S, f)LLTb(S,f).

For A→fA′A \rightarrow fA'A→fA′, f∈FIRST(fA′)f \in \text{FIRST}(fA')f∈FIRST(fA′), so we add A→fA′A \rightarrow fA'A→fA′ to the table cell LLTb(A,f)\mathrm{LLTb}(A, f)LLTb(A,f).

LL(1) Derivation​

LL table tells us, if we want a sequence xxx, how we should derive it.

  1. We append S\sout{S}S to xxx, then, we have a pointer at 000, call it iii. Now we have the intermediate sentence mmm. Initially, m=Sm = Sm=S.
  2. We check the table cell, LLTb(Leftmost(m),i@x)\mathrm{LLTb}(\text{Leftmost}(m), i@x)LLTb(Leftmost(m),i@x). We use the rule in this cell to perform leftmost derivation on the current intermediate sentence. If there exists no rule in the cell, it means that the given sequence cannot be derived from the grammar. If there exists no more non-terminals, the derivation is complete.
  3. Let i+ki + ki+k be the new pointer, then repeat 2. kkk is the length of the newly generated terminals on the left side.

We collect every derivation in the step 222. This is how we can use LL table to derive a sequence with leftmost derivation.

  1. We make x=fbbSx = fbb\sout{S}x=fbbS. Let pointer i=0i = 0i=0. Our initial sentence is SSS.
  2. We check the cell LLTb(S,f)\mathrm{LLTb}(S, f)LLTb(S,f). It tells to derive S⇒AS \Rightarrow AS⇒A.
