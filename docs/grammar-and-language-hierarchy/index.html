Grammar and Language Hierarchy

Grammar​

A grammar contains four parts,

  • Vocabulary: A set of symbols, noted as V\mathcal{V}V.
  • Non-Terminal: A set of symbols, noted as N\mathcal{N}N.
  • Rules: A set of rules, noted as R\mathcal{R}R.
  • Start Symbol: A symbol. Usually just SSS.

A grammar can generate a language.

Let's demonstrate one by one.

Vocabulary​

A vocabulary is a set of symbols. They are the vocabulary of the final sentence.

Non-Terminal​

The non-terminal is another set of symbols. We usually use capitalized letters for non-terminals. They are used to construct rules.

Rules​

A rule has two part, the left part is called the source, and the right part is called the target.

The source should be from the language (V+N)∗N(V+N)∗(\mathcal{V} + \mathcal{N})^*N(\mathcal{V} + \mathcal{N})^*(V+N)∗N(V+N)∗, to have at least one non-terminal, whereas the target should be from language (V+N)∗(\mathcal{V} + \mathcal{N})^*(V+N)∗.

We usually note a rule as,

source→target\text{source} \rightarrow \text{target}source→target

There must be a finite number of rules,

∣R∣<∞|\mathcal{R}| < \infty∣R∣<∞

Start Symbol​

Start symbol is just a non-terminal symbol. Usually just SSS.

Deduced Language​

A grammar is sufficient to deduce a language, which we call the deduced language, the L\mathcal{L}L, L⊂V∗\mathcal{L} \sub \mathcal{V}^*L⊂V∗.

We define a language called the intermediate language, the I\mathcal{I}I. It is a subset of the language (V+N)∗(\mathcal{V} + \mathcal{N})^*(V+N)∗.

Without condition, S∈IS \in \mathcal{I}S∈I. Sentence with only a start symbol is always a valid sentence in the intermediate language.

For any rule in the R\mathcal{R}R, if a sentence uuu is in the intermediate language set I\mathcal{I}I, and it has a substring that matches the source\text{source}source of the rule, then by replacing the source\text{source}source with target\text{target}target, we will have a new sentence vvv. We define v∈Iv \in \mathcal{I}v∈I.

We note this as,

v=sub(u,source,target)v = \text{sub}(u, \text{source},\text{target})v=sub(u,source,target)

This process from uuu to vvv is called derivation, noted as,

u⇒vu \Rightarrow vu⇒v

If the derivation repeats nnn times, we note,

u⇒nvu \Rightarrow^n vu⇒nv

For any times, use ∗*∗, for more than one times, use +^++.

u⇒∗vu \Rightarrow^* vu⇒∗v u⇒+vu \Rightarrow^+ vu⇒+v

That is to say,

I={u∣S⇒∗u}\mathcal{I} = \{u | S \Rightarrow^* u\}I={u∣S⇒∗u}

After we get I\mathcal{I}I,

L=I∩V∗\mathcal{L} = \mathcal{I} \cap \mathcal{V}^*L=I∩V∗

We choose,

To look for the language this grammar generates, we first get the complete set of intermediate language,

Initially,

I0={S}\mathcal{I_0} = \{S\}I0​={S}

The only applicable rule is S→ABS \rightarrow ABS→AB, thus,

I1={S,AB}\mathcal{I_1} = \{S, AB\}I1​={S,AB}

For ABABAB, it has two applicable rules,

AB→0ABB→1BAB \rightarrow 0 A \\ BB \rightarrow 1 BAB→0ABB→1B

Thus,

I2={S,AB,0A,1B}\mathcal{I_2} = \{S, AB, 0A, 1B\}I2​={S,AB,0A,1B}

For 0A0A0A, it has only one applicable rule,

There are no more applicable rules, thus, the language generated by this grammar is,

L=I∩V∗={0,01}\mathcal{L} = \mathcal{I} \cap \mathcal{V}^* = \{0, 01\}L=I∩V∗={0,01}

Chomsky Hierarchy​

Chomsky classify the grammars based on the form of rules. From lower types to higher types, there will be more and more restrictions on the rules, and the grammar will be simpler and simpler, so is the language.

In our book, we only talk about infinite languages. That is, ∣L∣=+∞|\mathcal{L}| = +\infty∣L∣=+∞ .

Type-0 Recursive Enumerable Grammars and Languages​

If a language can be generated by a recursive enumerable grammar, then it is of type-0 or a recursive enumerable language.

Type-0 does not enforce any constrain on the rules. All grammars are type-0. That is to say, all languages are type-0.

Type-1 Context Sensitive Grammars and Languages​

If a language can be generated by a context sensitive grammar, then it is of type-1 or a context sensitive language.

tip

Please note that if a language can be generated by type-nnn grammar, then it is also a type-nnn language. This mean that, for a language that generated from type-mmm where m>nm > nm>n, there is possibility that we chose a stupid grammar that can be simplified to a higher type. Thus type-mmm grammar doesn't always generates type-mmm language.

It enforces all rules to be of the form,

xAy→xzyx A y \rightarrow x z yxAy→xzy

Where ∣z∣>0|z| > 0∣z∣>0, but xxx and yyy can be empty, AAA is any non-terminal symbol.

info

For convenience, we sometimes allow a special rule,

S→λS \rightarrow \lambdaS→λ

If there exists no rules has target\text{target}target that contains SSS. This is called a nullable context sensitive language. The only different the language has from the context sensitive language is that allows λ∈L\lambda \in \mathcal{L}λ∈L.

This is obvious because we forbid SSS to exists on the right side, and thus if we remove S→λS \rightarrow \lambdaS→λ, we get a non-nullable context sensitive language that has no SSS in any sentence except for the single symbol sentence SSS.

After adding the rule S→λS \rightarrow \lambdaS→λ. We can only yield one new sentence λ\lambdaλ.

tip

Nullable languages allows null sentence and the rule S→λS \rightarrow \lambdaS→λ, but doesn't enforce it.

It is also true for the nullable regular language.

Type-2 Context Free Grammars and Languages​

If a language can be generated by a context free grammar, then it is of type-2 or a context free language.

A context free grammar requires,

A→xA \rightarrow xA→x

Where AAA is a non-terminal symbol, and xxx is any sentence.

info

Context free grammar allows null generation.

Type-3 Regular Grammars and Languages​

If a language can be generated by a regular grammar, then it is of type-3 or a regular language.

A regular grammar requires,

A→aBA \rightarrow aBA→aB

Or

A→aA \rightarrow aA→a

Where AAA and BBB are non-terminal symbols, and aaa is a symbol.

info

Nullable regular grammar adds,

S→λS \rightarrow \lambdaS→λ

And no rule yields a sentence with SSS.

This is the nullable regular language and nullable regular grammar.

Nullable regular language only differentiates itself from regular language in that it allows a null sentence.

tip

All finite languages are regular.
